Confidence Filter — Group D Row Drop

Group D
Method: Row filter on labeller_confidence_score ≥ 0.5

Records how confident the human annotator was when assigning a label. Left-skewed: annotators are more confident about normal cells than abnormal ones.

$$\text{keep row} \Leftrightarrow \text{labeller\_confidence\_score} \geq 0.5$$
Why threshold = 0.5? Rows below 0.5 are label-uncertain — training on them introduces systematic noise into the decision boundary while retaining the majority of data.
Also dropped from Group D (data leakage): cytodiffusion_anomaly_score and cytodiffusion_classification_confidence — produced by the same cytodiffusion system as anomaly_label. Their KDE shows 100% class separation: a textbook leakage signature.
2

IQR Trimming — Group C Outliers

Group C
Method: Tukey IQR Fences (multiplier 1.5) — row deletion
$$Q_1 = P_{25},\quad Q_3 = P_{75},\quad \text{IQR} = Q_3 - Q_1$$ $$\text{Lower fence} = Q_1 - 1.5\!\times\!\text{IQR}$$ $$\text{Upper fence} = Q_3 + 1.5\!\times\!\text{IQR}$$ $$\text{Remove row where } x_i < \text{Lower fence} \text{ or } x_i > \text{Upper fence}$$
Why delete rows (trimming) instead of capping?
Group C distributions are symmetric and near-Gaussian (see EDA chart). Extreme values are measurement errors or device noise — genuine corrupted observations, not biological outliers. Deleting them is correct; capping would keep corrupted values in the dataset.

Contrast Group A: Outliers there are real pathology signals → capped, not removed.

Physical Unit Normalization

Group B
Method: Magnification-aware conversion (px → µm)
$$\text{true\_cell\_area} = \frac{\text{cell\_area\_px}}{(\text{magnification\_x})^2}$$ $$\text{true\_perimeter} = \frac{\text{perimeter\_px}}{\text{magnification\_x}}$$
Pixels are device-dependent: a 300 px cell at 40× and at 100× represent the same physical cell. Area scales with the square of magnification (2-D), perimeter linearly. After conversion both features represent true µm measurements, comparable across all microscopes in the dataset.

magnification_x is dropped in Step 9 once it is no longer needed.
5

Staining Protocol Shifting

Group C Must run before Step 6
Method: Group-wise mean shift (per staining protocol)
Ordering constraint: Protocol shifting must run in Step 5, before Feature Engineering in Step 6. If chromaticity ratios were computed first (from raw colors), they would embed staining-protocol bias. By shifting colors first, ratios $R/(R+G+B)$ reflect the cell's true hue, not the choice of Giemsa vs. Wright stain.
For protocol $p$ and channel $c \in \{R,G,B,\text{intensity}\}$: $$\mu_c^{(p)} = \mathbb{E}[c \mid \text{protocol} = p]$$ $$\mu_c^{\text{global}} = \mathbb{E}[c]$$ $$x_{\text{shifted}} = x - \mu_c^{(p)} + \mu_c^{\text{global}}$$
Effect: Each protocol's color distribution is shifted to the global mean. The within-protocol variance is preserved; only the between-protocol offset is removed.

Fitted on X_train only — group means are stored, then applied to X_test without re-fitting, preventing leakage.
Affected features
mean_r mean_g mean_b stain_intensity

Feature Engineering — Domain Knowledge

Medical experts interpret ratios between raw measurements, not the numbers themselves. These five engineered features encode hematological domain knowledge. Color ratios are computed from protocol-shifted colors (Step 5).

Group A → Engineered1 — N/C Ratio (Nucleus-to-Cytoplasm)Gold Standard
$$\text{NC\_Ratio} = \frac{\text{nucleus\_area\_pct}}{100 - \text{nucleus\_area\_pct}}$$
The gold standard in hematology for identifying malignant cells. Abnormal / immature cells (blasts, cancer cells) have an enlarged nucleus that occupies most of the cell. NC Ratio = ∞ means the nucleus fills the entire cell; NC Ratio = 0 means no nucleus.

Source feature dropped after: nucleus_area_pct is a monotonic bijection of NC_Ratio → VIF = ∞ if both are retained.
Group A+B → Engineered2 — Form Factor (Shape Complexity)
$$\text{Form\_Factor} = \frac{4\pi \times \text{true\_cell\_area}}{\text{true\_perimeter}^2}$$
The isoperimetric quotient: 1.0 = perfect circle (normal morphology); lower values indicate elongated, lobulated, or membrane-deformed cells (abnormal). Mathematically, for a fixed area, a circle has the shortest possible perimeter — any deviation reduces this ratio.

Computed from physical-unit columns (true_cell_area, true_perimeter) created in Step 4 — not from raw pixel columns — so the ratio is magnification-invariant.

Note true_perimeter is dropped in Step 7: keeping true_cell_area, true_perimeter, and form_factor together gives VIF = ∞ for true_perimeter.
Group C → Engineered3 — Chromaticity Ratios (from protocol-shifted colors)
$$R_\text{ratio} = \frac{R_\text{shifted}}{R_\text{shifted}+G_\text{shifted}+B_\text{shifted}}$$ $$G_\text{ratio} = \frac{G_\text{shifted}}{R_\text{shifted}+G_\text{shifted}+B_\text{shifted}}$$
b_ratio is not kept: $R+G+B=1$ always, so $B = 1 - R - G$ — perfect linear dependency, VIF = ∞.
Raw RGB values change with lighting and stain concentration. Dividing by the total removes the brightness component, leaving only hue. Basophilic cells (blue) vs. eosinophilic cells (pink/red) become clearly separable in chromaticity space.

Computed after protocol shifting (Step 5) so that hue reflects cell biology, not staining methodology.
Group B+C → Engineered4 — Size Anomaly (Cell vs. Patient Baseline)
$$\text{size\_anomaly} = \frac{\text{cell\_diameter\_um}}{\text{mcv\_fl}}$$
A 12 µm cell is normal for one patient but giant for another whose average RBC is 7 µm. mcv_fl (mean corpuscular volume) captures the patient's own baseline. A high ratio flags cells that are anomalously large relative to that patient — a signal for blasts or macrophages.

Multicollinearity fix Both cell_diameter_um and mcv_fl are dropped in Step 7 after this feature is computed. Keeping a divisor alongside its ratio (size_anomaly = cell_diameter_um / mcv_fl) inflates VIF for all three — the ratio captures the cross-group interaction; the raw values add no independent signal.
Group B → Engineered5 — Physical-unit Features (from Step 4)

true_cell_area produced in Step 4 is carried through as an engineered feature. It replaces cell_area_px (dropped in Step 7).
true_perimeter is also produced in Step 4 and is used by Step 6 to compute form_factor, then dropped in Step 7: since $\text{form\_factor} = \frac{4\pi \cdot \text{true\_cell\_area}}{\text{true\_perimeter}^2}$, keeping all three creates a perfect linear dependency (VIF = ∞ for true_perimeter).

Drop Source Features — Multicollinearity Prevention

New Step
Rationale: features now fully encoded by engineered ones must be removed
Keeping both a raw feature and its monotonic transformation guarantees perfect multicollinearity (VIF = ∞), which causes:
• Inflated coefficient variance in Logistic Regression and other linear models
• Numerical instability (singular matrices)
• Wasted capacity in the ColumnTransformer and VIF filter (which would drop them anyway, but explicitly removing them here is cleaner and faster)

Per-Group Numeric Scaling (ColumnTransformer)

A ColumnTransformer applies a different sub-pipeline to each feature group based on its statistical profile. Each group's scaler is justified by EDA distribution analysis.

Group A Pathology Signals → Winsorization [1,99%] → Yeo-Johnson → RobustScaler
Features after Step 7
eccentricity lobularity_score chromatin_density nucleus_area_pct ✕ circularity
EDA: heavy right skew, well-separated KDEs — pathological outliers present and meaningful
1 · Winsorization
$$x' = \begin{cases} P_1 & x < P_1 \\ x & P_1 \le x \le P_{99} \\ P_{99} & x > P_{99} \end{cases}$$

Why cap not trim: Extreme values here ARE real pathology signals. Capping bounds them so they don't dominate linear models, while preserving the information that they are extreme.

2 · Yeo-Johnson Transform
$$\psi(x,\!\lambda)\!=\!\begin{cases}\frac{(x+1)^\lambda-1}{\lambda} & x\!\ge\!0,\lambda\!\ne\!0\\\ln(x+1) & x\!\ge\!0,\lambda\!=\!0\\\frac{-((-x+1)^{2-\lambda}-1)}{2-\lambda} & x\!<\!0,\lambda\!\ne\!2\\-\ln(-x+1) & x\!<\!0,\lambda\!=\!2\end{cases}$$

Why not Box-Cox: Box-Cox requires $x > 0$ strictly. Yeo-Johnson handles non-positive values — needed since features may cross zero after centering. $\lambda$ is optimised via MLE per feature.

3 · RobustScaler
$$x' = \frac{x - \text{median}(x)}{\text{IQR}(x)}$$

Why not StandardScaler: StandardScaler uses mean and σ — both non-robust to outliers. Median and IQR are breakdown-point ≥ 25%, making RobustScaler resistant to residual extremes after capping.

Group B Morphology → Winsorization [1,99%] → QuantileTransformer (uniform) → StandardScaler
Features after Step 7
cell_diameter_um ✕ cell_area_px ✕ perimeter_px ✕ cytoplasm_ratio membrane_smoothness granularity_score
EDA: multi-modal distributions — multiple peaks caused by different cell sub-populations in one dataset
QuantileTransformer (uniform output)
$$x' = F_{\text{uniform}}^{-1}\!\bigl(F_{\text{empirical}}(x)\bigr)$$

Maps each value to its empirical quantile rank, then maps that rank to $\mathcal{U}(0,1)$. The result: each feature is flattened to a uniform distribution regardless of original shape — multi-modal peaks and skewness are both resolved in one step.

Why QuantileTransformer over Yeo-Johnson for Group B?
Yeo-Johnson reduces skewness but cannot handle multimodality — it would leave the multiple peaks intact. QuantileTransformer's rank-based mapping treats every quantile equally, producing a flat distribution that allows models to find a consistent decision boundary across all sub-populations.
Group C Hematology / Noise → MinMaxScaler only
Features after Step 7
wbc_count_per_ul rbc_count_millions_per_ul hemoglobin_g_dl hematocrit_pct platelet_count_per_ul mcv_fl ✕ mchc_g_dl mean_r / g / b ✕ stain_intensity
EDA: near-Gaussian, symmetric, KDEs for label 0/1 completely overlapping
MinMaxScaler
$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}} \in [0,1]$$

Simplest appropriate scaler: maps all values to [0,1] preserving the original distribution shape.

Why the simplest scaler?
Group C is already near-Gaussian after IQR trimming — no skewness to fix. Outliers have been removed. A power transform would be overkill and could distort the (already clean) Gaussian shape. These features have near-zero discriminative power and will largely be removed by feature selection (Step 14).
Engineered Engineered Features → StandardScaler
nc_ratio form_factor r_ratio g_ratio b_ratio ✕ size_anomaly true_cell_area true_perimeter ✕

true_perimeter is dropped in Step 7 (VIF = ∞ with form_factor + true_cell_area)

$$x' = \frac{x - \mu}{\sigma}$$

Engineered features are derived quantities with no strong prior outlier concern after Steps 1–7. Z-score normalisation centres them without any distribution assumptions.

GMM Clustering — Multi-modality Handling

Algorithm: Gaussian Mixture Models, optimal K via BIC in [3, 5]
GMM density: $$p(\mathbf{x}) = \sum_{k=1}^{K} \pi_k\,\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)$$ $$\pi_k \ge 0,\quad \sum_k \pi_k = 1$$ Optimal K via BIC (lower is better): $$\text{BIC} = k\ln n - 2\ln\hat{\mathcal{L}}$$
Why GMM over K-Means?
K-MeansGMM
AssignmentHard (1 cluster)Soft (probability)
Cluster shapeSpherical onlyEllipsoidal
Overlap allowedNoYes
Multi-modal dataLimitedIdeal
anomaly_label is never passed to GMM — clustering is unsupervised on morphological features only, preventing leakage.

Gaussian Noise Augmentation

Group B
Method: Additive Gaussian noise on morphological features ($\sigma$ = 1% of std)
$$x'_i = x_i + \varepsilon_i,\quad \varepsilon_i \sim \mathcal{N}(0,\,\sigma^2)$$ $$\sigma = 0.01 \times \text{std}(x)$$
Simulates realistic measurement noise from microscope cameras and hematological analyzers without distorting biological signals. At 1% σ, the perturbation is undetectable visually but prevents the model from memorising exact sensor readings (overfitting to device precision).

Targets updated after Step 7 drops — cell_area_px, perimeter_px, mean_r/g/b, cell_diameter_um, mcv_fl, true_perimeter no longer exist:
cell_diameter_um ✕ cytoplasm_ratio membrane_smoothness granularity_score true_cell_area true_perimeter ✕ stain_intensity
Group A is excluded: pathological signals in Group A are precious — any noise could corrupt the very features most critical for classification.

Covariate Shift Simulation

Group C
Method: Multiplicative random scaling factor $\sim\mathcal{U}(0.98, 1.02)$
$$x'_i = x_i \times f_i,\quad f_i \sim \mathcal{U}(0.98,\,1.02)$$
In real clinical settings, hematological analyzers at different hospitals are calibrated slightly differently — Hospital A's hemoglobin meter may consistently read ~1% higher than Hospital B's. This 2% multiplicative perturbation simulates that inter-hospital instrument variability, making the model robust to deployment drift.

Feature Selection — 4-Stage Sequential Filter

Stage 1VIF — Multicollinearity (threshold 10)
$$\text{VIF}_j = \frac{1}{1 - R^2_j}$$
VIF = 10 means $R^2 = 0.9$: 90% of the feature's variance is explained by others. Removing such features stabilises Logistic Regression coefficients and reduces numerical issues in matrix inversion.
Stage 2Pearson Correlation Filter (threshold 0.9)
$$r_{xy} = \frac{\sum(x_i-\bar{x})(y_i-\bar{y})}{\sqrt{\sum(x_i-\bar{x})^2\sum(y_i-\bar{y})^2}}$$
Catches pairwise linear redundancy not exposed by VIF (which is multivariate). For any pair with $|r|>0.9$, one is discarded.
Stage 3Mutual Information — SelectKBest (k = 20)
$$I(X;Y) = \sum_{x,y} p(x,y)\log\frac{p(x,y)}{p(x)p(y)}$$
Unlike Pearson $r$, MI captures non-linear dependencies. Group C features (hematological counts) have near-zero MI with the target → eliminated. Group A features score highest.
Stage 4RFECV — Recursive Elimination + 5-fold CV
  1. Fit RandomForest on remaining features
  2. Rank by $w_j$ (feature importance)
  3. Remove feature with lowest $w_j$
  4. Repeat; pick best count via 5-fold F1 CV
MI ranks features independently (univariate). RFECV detects synergy: a feature weak alone but powerful with another is kept. Expected final count: 10–12.
15

SMOTE-Tomek — Class Imbalance

Synthetic oversampling + boundary undersampling
Problem: skewness = 0.77 → majority-class bias without intervention.
SMOTE
$$\mathbf{x}_\text{new} = \mathbf{x}_i + \lambda(\mathbf{x}_j - \mathbf{x}_i),\quad \lambda\sim\mathcal{U}(0,1)$$

Interpolates between minority sample $\mathbf{x}_i$ and a random KNN neighbour $\mathbf{x}_j$. Creates new synthetic samples rather than duplicating existing ones.

Tomek Links
$$(\mathbf{x}_i,\mathbf{x}_j)\text{ is a Tomek Link if each is the other's nearest neighbour and }\text{class}(i)\ne\text{class}(j)$$

Remove the majority-class member of each link.

Cleans boundary ambiguity — removes the majority samples closest to the minority boundary, creating a clean margin.

SMOTE-Tomek is applied only during training (fitted only on X_train inside the pipeline). The test set is never modified — evaluating on real unaugmented samples ensures honest performance metrics.

Model Hyperparameter Choices

All hyperparameters are fixed (no grid search) and are read from config.yml → model_training.models[*].params. Below is the justification for each non-default value.

Decision Tree
ParameterValueWhy
max_depth10Prevents over-deep trees from memorising noise; 10 levels captures complex interactions without overfitting the small dataset.
min_samples_split5A node needs ≥5 samples to be split — prevents splits on individual outlier points.
min_samples_leaf2Each leaf must contain ≥2 samples, eliminating singleton leaves that memorise noise.
class_weightbalancedWeights each class inversely proportional to its frequency → compensates for class skewness 0.77.
max_featuressqrtUses $\sqrt{n_\text{features}}$ features per split — standard variance reduction technique.
ccp_alpha0.0No post-pruning; VIF + RFECV already removed redundant features, so additional pruning is not needed.
Gaussian Naive Bayes
ParameterValueWhy
var_smoothing1e-9Adds a small value to estimated per-feature variance to avoid division by zero when a feature has very low variance post-scaling. Default sklearn value, appropriate here since Yeo-Johnson has already stabilised variances.
GaussianNB assumes features follow $\mathcal{N}(\mu_k, \sigma_k^2)$ per class $k$. This assumption is satisfied for Group A features after Yeo-Johnson transform. Class-conditional likelihood: $$p(x_j \mid y=k) = \frac{1}{\sqrt{2\pi\sigma_{jk}^2}}\exp\!\left(-\frac{(x_j-\mu_{jk})^2}{2\sigma_{jk}^2}\right)$$
K-Nearest Neighbours
ParameterValueWhy
n_neighbors5Classic starting point. $k=1$ is too noisy; very large $k$ over-smooths the boundary. 5 balances variance and bias for ~4700 training samples.
weightsdistanceCloser neighbours contribute more. Important for imbalanced data — a minority sample near the boundary gets fairer treatment than uniform weighting.
metric / pminkowski / 2Euclidean distance — appropriate for scaled numerical features where all axes have comparable units post-scaling.
Logistic Regression
ParameterValueWhy
C1.0Inverse regularisation strength. C=1 is the neutral (default) regularisation — neither too sparse nor too flexible. Strong regularisation would be premature without tuning.
solversagaConfig says lbfgs but the code overrides to saga — the only solver supporting both L1 and L2 penalties simultaneously, enabling full GridSearch over penalty type.
max_iter5000Increased from default 100 — needed after extensive preprocessing (more orthogonal features take longer to converge).
class_weightbalancedAdjusts loss weights for class imbalance, same rationale as Decision Tree.
Random Forest
ParameterValueWhy
n_estimators100100 trees gives excellent bias-variance tradeoff at moderate compute cost. Forest variance is $\propto 1/n_\text{trees}$ so gains diminish rapidly beyond 100.
max_depth10Same as DT — prevents any individual tree from overfitting (though the ensemble averages this out, limiting depth speeds up training).
max_featuressqrtDecorrelates trees: each split considers only $\sqrt{n}$ random features, so trees make different mistakes and averaging reduces variance.
class_weightbalancedEach tree is weighted per class frequency — handles imbalance at the tree level, not just the dataset level.
bootstraptrueBagging: each tree is trained on a bootstrap sample (≈63% unique samples) — the held-out OOB samples provide a free validation signal.
XGBoost
ParameterValueWhy
n_estimators100100 boosting rounds. Unlike RF, boosting is sequential so more trees = more risk of overfitting; 100 is the standard starting point.
max_depth6Shallower than RF (6 vs. 10) — in boosting each tree corrects residuals of the previous, so shallow "weak learners" are preferable to deep ones.
learning_rate0.1Conservative shrinkage: each tree contributes only 10% of its prediction. Lower rate = more robust to overfitting, at the cost of needing more trees.
subsample0.8Uses 80% of training data per tree (stochastic gradient boosting). Introduces variance reduction similar to bagging.
colsample_bytree0.8Uses 80% of features per tree — mirrors RF's max_features for tree decorrelation.
scale_pos_weight3$\approx n_\text{negative}/n_\text{positive}$. For skewness=0.77, the majority class is ~3× the minority → this parameter tells XGBoost to penalise minority-class errors 3× more.
